# 威胁模型

## 目标

威胁模型用于定义项目能保证什么、不能保证什么，以及证明报告需要覆盖哪些攻击面。没有明确威胁模型，就不存在可验证的隐私保证。

## 保护资产

需要保护的资产包括：

- 原始输入数据。
- 直接标识符，如姓名、手机号、身份证号、邮箱、客户号和设备 ID。
- 准标识符，如年龄、地区、职业、日期、机构、金额和罕见属性。
- 敏感属性，如疾病、财务状态、法律事项、交易行为和安全事件。
- 本地密钥。
- token 映射表。
- 策略配置。
- 原始日志、trace、缓存和向量索引。

## 默认攻击者

默认攻击者具有以下能力：

- 可以看到发送给外部 LLM API 的完整外部可见视图。
- 可以看到外部模型返回内容。
- 可以保留多轮请求之间的上下文。
- 可以使用公开数据进行链接攻击。
- 可以尝试通过 prompt injection 诱导系统泄露更多信息。
- 可以利用频率、格式、实体关系和上下文暗示推断隐私。

默认攻击者不具备以下能力：

- 读取本地密钥。
- 读取本地 token 映射表。
- 读取原始数据。
- 修改本地策略引擎。
- 绕过网关直接调用外部 API。

## 信任假设

本项目默认信任：

- 本地运行环境。
- 本地密钥管理。
- 本地策略引擎。
- 本地证明报告生成器。
- 本地审计存储。

如果这些组件被攻破，项目默认保证不再成立。此类场景需要额外的操作系统隔离、硬件隔离、访问控制、密钥轮换和运行时防护。

## 主要威胁

### 直接恢复

攻击者尝试从 token 或占位符恢复原始姓名、手机号、身份证号等。该风险由 HMAC tokenization、格式保持加密或随机 tokenization 约束。

### 频率分析

确定性 token 会保留相等性，也会暴露出现频率。若高频 token 对应可猜测实体，可能产生泄露。策略应声明 equality leakage 和 frequency leakage。

### 链接攻击

攻击者利用公开数据，将脱敏后的准标识符组合与真实个体匹配。例如：

```text
城市 + 职业 + 日期 + 罕见疾病
```

该风险需要通过泛化、抑制、分桶、差分隐私或本地处理降低。

### 上下文推断

文本中可能没有显式身份证号，但包含足以识别实体的上下文：

```text
徐汇区某医院上周接收的罕见病患者
```

此类风险无法由正则表达式完全捕获，需要任务约束和外部可见信息最小化。

### 多轮拼接

攻击者可能通过多轮交互逐步获得不同片段，最终拼接出身份。网关需要维护会话级隐私预算和上下文风险。

### 工具调用泄露

Agent 工具可能返回未脱敏字段，例如数据库查询结果、日志片段或文件内容。所有工具输入输出都必须经过同一策略引擎。

### 输出再泄露

外部模型可能在输出中重构、猜测或编造敏感信息。输出检查层需要阻断未授权敏感内容。

### 日志和缓存泄露

即使 prompt 已脱敏，原始数据仍可能残留在本地日志、远程日志、trace、缓存或向量库中。审计系统必须记录存储边界。

## 非目标

本项目默认不解决：

- 本地密钥泄露后的保护。
- 恶意管理员读取原始数据。
- 任意自由文本中所有隐私含义的完备识别。
- 外部世界知识足够强时的无条件不可重识别。
- 下游任务本身要求输出身份信息时的隐私保护。
- 用户绕过网关直接调用外部 API。

## 风险声明

脱敏有效性不是单个模型输出的属性，而是机制、策略、输入空间、任务集合和威胁模型共同决定的属性。证明报告必须记录这些前提，否则报告不可解释。
